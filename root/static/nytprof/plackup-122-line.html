14=head1 NAME
15
16plackup - Run PSGI application with Plack servers
17
18=head1 SYNOPSIS
19
20 # read your app from app.psgi file
21 plackup
22
23 # choose .psgi file from ARGV[0] (or with -a option)
24 plackup hello.psgi
25
26 # switch server implementation with --server (or -s)
27 plackup --server HTTP::Server::Simple --port 9090 --host 127.0.0.1 test.psgi
28
29 # use UNIX socket to run FCGI daemon
30 plackup -s FCGI --listen /tmp/fcgi.sock myapp.psgi
31
32 # launch FCGI external server on port 9090
33 plackup -s FCGI --port 9090
34
35=head1 DESCRIPTION
36
37plackup is a command line utility to run PSGI applications from the command
38line.
39
40plackup automatically figures out the environment it is run in, and
41runs your application in that environment. FastCGI, CGI, AnyEvent and
42others can all be detected. See L<Plack::Loader> for the authoritative
43list.
44
45C<plackup> assumes you have an C<app.psgi> script in your current directory.
46The last statement of C<app.psgi> should be a code reference that is a PSGI
47application:
48
49 #!/usr/bin/perl
50 use MyApp;
51 my $application = MyApp->new;
52 my $app = sub { $application->run_psgi(@_) };
53
54=head1 ARGUMENTS
55
56=over 4
57
58=item .psgi
59
60 plackup --host 127.0.0.1 --port 9090 /path/to/app.psgi
61
62The first non-option argument is used as a C<.psgi> file path. You can
63also set this path with C<-a> or C<--app>. If omitted, the
64default file path is C<app.psgi> in the current directory.
65
66=back
67
68=head1 OPTIONS
69
70=over 4
71
72=item -a, --app
73
74Specifies the full path to a C<.psgi> script. You may alternately provide this
75path as the first argument to C<plackup>.
76
77=item -e
78
79Evaluates the given perl code as a PSGI app, much like perl's C<-e>
80option:
81
82 plackup -e 'sub { my $env = shift; return [ ... ] }'
83
84It is also handy when you want to run a custom application like Plack::App::*.
85
86 plackup -MPlack::App::File -e 'Plack::App::File->new(...)->to_app'
87
88You can also specify C<-e> option with C<.psgi> file path to wrap the
89application with middleware configuration from the command line. You
90can also use L<Plack::Builder> DSL syntax inside C<-e> code. For example:
91
92 plackup -e 'enable "Auth::Basic", authenticator => ...;' myapp.psgi
93
94is equivalent to the PSGI application:
95
96 use Plack::Builder;
97 use Plack::Util;
98
99 builder {
100 enable "Auth::Basic", authenticator => ...;
101 Plack::Util::load_psgi("myapp.psgi");
102 };
103
104Note that when you use C<-e> option to enable middleware, plackup
105doesn't assume the implicit C<app.psgi> path. You must either pass the
106path to your C<.psgi> file in the command line arguments or load the
107application inside C<-e> after the C<enable>.
108
109 plackup # Runs app.psgi
110 plackup -e 'enable "Foo"' # Doesn't work!
111 plackup -e 'enable "Foo"' app.psgi # Works
112 plackup -e 'enable "Foo"; sub { ... }' # Works
113
114=item -o, --host
115
116Binds to a TCP interface. Defaults to undef, which lets most server backends
117bind to the any (*) interface. This option is only valid for servers which support
118TCP sockets.
119
120=item -p, --port
121
122Binds to a TCP port. Defaults to 5000. This option is only valid for servers
123which support TCP sockets.
124
125=item -s, --server, the C<PLACK_SERVER> environment variable
126
127Selects a specific server implementation to run on. When provided, the C<-s> or
128C<--server> flag will be preferred over the environment variable.
129
130If no option is given, plackup will try to detect the I<best> server
131implementation based on the environment variables as well as modules loaded by
132your application in C<%INC>. See L<Plack::Loader> for details.
133
134=item -S, --socket
135
136Listens on a UNIX domain socket path. Defaults to undef. This option is only
137valid for servers which support UNIX sockets.
138
139=item -l, --listen
140
141Listens on one or more addresses, whether "HOST:PORT", ":PORT", or "PATH"
142(without colons). You may use this option multiple times to listen on multiple
143addresses, but the server will decide whether it supports multiple interfaces.
144
145=item -D, --daemonize
146
147Makes the process run in the background. It's up to the backend server/handler
148implementation whether this option is respected or not.
149
150=item -I
151
152Specifies Perl library include paths, like C<perl>'s -I option. You may add
153multiple paths by using this option multiple times.
154
155=item -M
156
157Loads the named modules before loading the app's code. You may load multiple
158modules by using this option multiple times.
159
160=item -E, --env, the C<PLACK_ENV> environment variable.
161
162Specifies the environment option. Setting this value with C<-E> or C<--env>
163also writes to the C<PLACK_ENV> environment variable. This allows applications
164or frameworks to tell which environment setting the application is running on.
165

200=item -L, --loader
201

208=item --access-log
