360=item load_class
361
362 my $class = Plack::Util::load_class($class [, $prefix ]);
363
364Constructs a class name and C<require> the class. Throws an exception
365if the .pm file for the class is not found, just with the built-in
366C<require>.
367
368If C<$prefix> is set, the class name is prepended to the C<$class>
369unless C<$class> begins with C<+> sign, which means the class name is
370already fully qualified.
371
372 my $class = Plack::Util::load_class("Foo"); # Foo
373 my $class = Plack::Util::load_class("Baz", "Foo::Bar"); # Foo::Bar::Baz
374 my $class = Plack::Util::load_class("+XYZ::ZZZ", "Foo::Bar"); # XYZ::ZZZ
375
376Note that this function doesn't validate (or "sanitize") the passed
377string, hence if you pass a user input to this function (which is an
378insecure thing to do in the first place) it might lead to unexpected
379behavior of loading files outside your C<@INC> path. If you want a
380generic module loading function, you should check out CPAN modules
381such as L<Module::Runtime>.

424=item load_psgi
425
426 my $app = Plack::Util::load_psgi $psgi_file_or_class;
427
428Load C<app.psgi> file or a class name (like C<MyApp::PSGI>) and
429require the file to get PSGI application handler. If the file can't be
430loaded (e.g. file doesn't exist or has a perl syntax error), it will
431throw an exception.
432
433B<Security>: If you give this function a class name or module name
434that is loadable from your system, it will load the module. This could
435lead to a security hole:
436
437 my $psgi = ...; # user-input: consider "Moose.pm"
438 $app = Plack::Util::load_psgi($psgi); # this does 'require "Moose.pm"'!
439
440Generally speaking, passing an external input to this function is
441considered very insecure. But if you really want to do that, be sure
442to validate the argument passed to this function. Also, if you do not
443want to accept an arbitrary class name but only load from a file path,
444make sure that the argument C<$psgi_file_or_class> begins with C</> so
445that Perl's built-in do function won't search the include path.
